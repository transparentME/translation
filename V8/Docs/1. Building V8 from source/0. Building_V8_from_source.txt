Building V8 from source
构建v8

In order to be able to build V8 from scratch on Windows/Linux/macOS for x64, please follow the following steps.
根据以下操作，就可以在64位的Windows/Linux/MacOs的操作系统上构建V8啦～
Getting the V8 source code
获取V8源码
Follow the instructions in our guide on checking out the V8 source code.
按[获取v8源码](https://v8.dev/docs/source-code)的指引来做吧～

Installing build dependencies
安装构建所需的依赖
For macOS: install Xcode and accept its license agreement. (If you’ve installed the command-line tools separately, remove them first.)
使用macOS用户: 安装Xcode并接受它的许可证协议。（如果你已经单独安装了命令行工具，可以看看[这个帖子](https://bugs.chromium.org/p/chromium/issues/detail?id=729990#c1)的内容）
Make sure that you are in the V8 source directory. If you followed every step in the previous section, you’re already at the right location.
确保你当前在v8源目录中。如果你跟着之前步骤一步一步来的呢，那你现在就在你应该在的位置啦～

Download all the build dependencies:
下载所有构建所需的依赖：
执行: gclient sync

For Googlers - If you see Failed to fetch file or Login required errors when running the hooks, try authenticating with Google Storage first by running:
Googler们：当运行钩子的时候，如果你获取文件失败了，或者登陆失败，那就先运行以下操作，在Google Storage身份验证：

执行: gsutil.py config

Login with your @google.com account, and enter 0 when asked for a project ID.
登陆你的@google.com账号，输入projectID的值0；

This step is only needed on Linux. Install additional build dependencies:
只有Linux系统需要操作这一步。安装额外的构建依赖：
执行: ./build/install-build-deps.sh

Building V8
构建v8
1. Make sure that you are in the V8 source directory on the master branch.
第一步：进入v8源项目的master分支

执行: cd /path/to/v8

2. Pull in the latest changes and install any new build dependencies:
第二步: 拉取最新的修并安装所有依赖

执行:git pull && gclient sync

3. Compile the source:
第三步: 编译源码：

执行: tools/dev/gm.py x64.release

Or, to compile the source and immediately run the tests:
或者，编译源码后，立即运行测试：

tools/dev/gm.py x64.release.check
For more information on the gm.py helper script and the commands it triggers, see Building with GN.
想了解更多关于gm.py的帮助脚本以及它所触发执行的命令，可以参考[GN构建](https://v8.dev/docs/build-gn)一文。
